How It Works

LSB Steganography Defined

Digital images are composed of millions of pixels. In a standard 24-bit image, each pixel uses 8 bits (one byte) for Red, 8 bits for Green, and 8 bits for Blue. This means each color channel has a value from 0 to 255.

The **Least Significant Bit (LSB)** is the very last bit in that byte (the 8th bit). It controls microscopic variations in a color's shade. Altering this bit changes the color value by only +1 or -1, which is **completely imperceptible to the human eye**.

Analogy: Imagine a bucket filled with exactly 10,000 grains of sand. Steganography is like adding or removing a single grain. The bucket still looks exactly the same, but the total count has been modified to hold data.

Our Extraction Process

This script focuses specifically on the **Blue** channel of every pixel. We use the HTML5 Canvas API to access the raw binary data of the image. The algorithm is straightforward:

  1. It loops through every single pixel in the image sequentially.
  2. For each pixel, it isolates the 8-bit byte representing the **Blue** value.
  3. It performs a bitwise operation (blueByte & 1) to extract only the 8th bit (the LSB).
  4. It collects these individual bits. Every 8 bits collected form a new byte.
  5. This new byte is then translated back into its equivalent ASCII character (e.g., 01001001 becomes 'I').

When pieced together, this hidden binary stream reconstructs the secret plaintext message, which is then revealed on the screen.

A true steganographic analysis would require special software, but this JavaScript lab demonstrates the exact underlying principle that digital forensic tools automate.
